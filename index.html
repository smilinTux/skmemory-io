Memory, Reinvented

💾 Persistent Across Sessions

Memories survive restarts, crashes, and provider switches. Your agent picks up exactly where it left off — every time.

🌐 Cross-Platform Unity

Same memory in Cursor, VS Code, Terminal, Neovim, mobile. One agent, one memory store, every platform just a window into the same mind.

🔐 GPG Encrypted at Rest

Every memory entry encrypted with your CapAuth PGP key. No one reads your agent's memories but you. Not even us.

🔄 Syncthing P2P Sync

Memories propagate across all your devices via encrypted P2P mesh. No cloud middleman. Laptop, server, phone — always in sync.

📊 Structured & Searchable

Typed memory entries with tags, categories, and timestamps. Your agent searches its own history to give you better answers.

🛡️ Audit Trail

Every memory access logged by SKSecurity. Know exactly what was remembered, when, and by whom. Full accountability.

How It Works

~/.skcapstone/memory/
├── sessions/ # Per-session memory snapshots
├── long_term/ # Persistent cross-session memories
├── preferences/ # User preferences & patterns
├── skills/ # Learned capabilities
└── index.json # Searchable memory index
# Your agent remembers you told it about your project structure
skcapstone memory search "project architecture"

# It finds the memory from 3 weeks ago, on a different machine
Found: [2026-02-01] "Project uses FastAPI + SQLModel, deployed on SKStacks"
Tags: architecture, stack, deployment
Source: cursor-session-feb01 (laptop)

# Push memories to the P2P mesh
skcapstone sync push
# All devices now have your latest context

Get Started

Your agent's memory starts here.

# Your agent remembers from now on
skcapstone status
